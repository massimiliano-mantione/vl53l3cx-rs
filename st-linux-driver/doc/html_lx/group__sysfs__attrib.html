Detailed Description

stmvl53lx can be controled via a set of exposed as sysf attribute under /sys/class/inputx/attribe_name where x is the input device number associated with the vl53lx device . X is a dynamic number that depend on input device present in the system it can changes, depending order of driver loaded, hot plug device like usb mice and keyboard.

sysfs attribute "enable_ps_sensor" [rd/wr]

  • read show the current enable state
  • write set the new state value "0" put sensor off "1" put it on
Returns
0 on success , EINVAL if fail to start

their's no check and assume exclusive usage of sysfs and ioctl
Sensor will be put on/off disregard of any setup done by the ioctl channel.

sysfs attribute "roi" [rd/wr]

  • read show the current user customized roi setting
  • write set user custom roi, it can only be done while not ranging.

syntax for set input roi

  • "tlx tly brx bry\n" will set the roi
Warning
roi coordinate is not image x,y(down) but euclidian x,y(up)
roi validity is only check at next range start
1 >#to set roi
2 >echo "0 15 15 0" > /sys/class/input6/roi
3 >echo $?
4 0
5 >cat /sys/class/input6/roi
6 "0 15 15 0"
7 >echo "1" > /sys/class/input6/enable_ps_sensor
8 #try to set roi while ranging
9 >echo "0 15 15 0" > /sys/class/input6/roi
10 [58451.912109] stmvl53lx_store_roi: ERROR can't set roi while ranging
11 >echo $?
12 1
